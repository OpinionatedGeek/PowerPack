Pre-Requisites Of The ASP.NET PowerPack

The PowerPack requires ASP.NET v2.0.

Installing The ASP.NET PowerPack

Installing the ASP.NET PowerPack should be quite straightforward. We’ve chosen not to provide a Setup.exe, (see our reasons why) so all files are provided in a ZIP file. New for .NET 2.0, the only file required to add the ASP.NET PowerPack functionality is the DLL - all client-side scripts and images are now embedded resources inside the DLL.

The ZIP file should contain: 

	- /bin
		- OpinionatedGeek.Web.PowerPack.dll
	- /Samples
		- Some sample ASPX pages demonstrating the controls
	- OpinionatedGeek ASP.NET PowerPack.chm (help file)
	- Readme.html (this document)

The /Samples folder is not necessary for the running of the ASP.NET PowerPack, and so can be removed at any time. It is only there to provide some help in showing how to run the controls on an ASPX page.

Similarly, the OpinionatedGeek ASP.NET PowerPack.chm is a help file for developers to use when creating web solutions. It is also not necessary for the general running of the controls on a web site.

However, it is critical that the OpinionatedGeek.Web.PowerPack.dll is placed in your application root’s /bin folder. This is necessary for the ASP.NET system to find the controls at run time.

Running The Samples

Assuming you unzipped all the files to the web root on your local machine, you should be able to access all the samples just by going to http://localhost/Samples/.

Licensing

The OpinionatedGeek ASP.NET PowerPack is licensed at no cost for local use only. There is no cost for using it during development, testing or staging, as long as the server is accessed through its local name. If the site is accessed through a fully-qualified name or IP address, you need to buy a license.

In a nutshell: if you use a ‘.’ in the server name in the URL, you need to pay for a license.

Some examples:

  • http://localhost/ - free.
  • http://127.0.0.1/ - free.
  • http://myserver/ - free.
  • http://www.mydomain.com/ - you must pay for a license.
  • http://staging.mydomain.com/ - you must pay for a license.
  • http://111.111.111.111/ - you must pay for a license.

Licenses are available for individual web sites, named web servers, or for embedding the ASP.NET PowerPack in applications. You can buy licenses for the whole ASP.NET PowerPack or just the controls you want to use.

You can buy a license at any time by visiting our online store.

Upgrades And Patches

You have downloaded a 2.x version of the ASP.NET PowerPack. It is planned that all updates to 2.x will be similarly available - you should just be able to download them from OpinionatedGeek.com, just as you downloaded this version.

If you have bought a license for the ASP.NET PowerPack, it will work with all future 1.x versions and upgrades, so again you are free to download updated versions of the ASP.NET PowerPack from OpinionatedGeek.com as and when they become available.

No decisions have yet been made on version 3.x of the ASP.NET PowerPack, how it will be licensed, whether it will be downloadable the way version 2.x is, whether 2.x licenses will continue to work with 3.x etc. Really, 3.x is too far away to make any decisions about yet.

Iron-Clad Guarantee

Unlike some companies who subtly change the license terms in patch and support releases, you have our iron-clad guarantee that the licensing terms for version 1.x will never be changed to your detriment. You can buy the ASP.NET PowerPack safe in the knowledge that there are no hidden tricks to snare you.

Further Help

The best, detailed help on each control and how it should be used is contained in the compiled help file ‘OpinionatedGeek ASP.NET PowerPack.chm’. It contains full details of the inheritance hierarchy, as well as descriptions of every public and protected property and method.

If you have questions about what a method does, or how to specify an attribute in a custom tag, this detailed help file should be your first destination.
